Notes

You can use Ans with "." only if Ans is an integer → if Ans is 3 and Ans.4 is evaluated, then 3.4 will be outputted; if Ans is a decimal a period will not be able to be placed
Multifactorialing is not allowed → 6!! will not evaluate to neither 6×4×2 nor (6!)! but will instead error
All buttons with a black border around them are able to be held for 1 second to display a popup → with the Sx and log10(x) buttons, pressing the 'a' key within the inputs will set it to 'Ans()' where the value inside the brackets accesses the entry history to return the according value
The calculator may repeat 9's or 0's when it shouldn't → due to floating point precision issues which are inherent in code, outputs such as 0.30000000000000004 or 0.29999999999999999 may occur, although their effect on the desired value is negligible
There are three possible errors to receive → CalculationError (most common) when trying to evaluate an invalid expression, ZeroDivisionError when trying to divide by 0, and EntryError when either trying to evaluate when out of space, or using an invalid Ans() value for S or log()
Hovering over an entry inputted value in the top left will display the replacement value used → hovering over the Ans(2) in LOGAns(2) (after inputting that to the log base popup) will reveal the second answer entry
Implicit multiplication is valid → having '3π' or '3sin(90)' or '3!3' will be interpreted as '3*π', '3*sin(90)', and '3!*3'
The approximate number of vacant entries is calculated based on the average byte size of each entry (each character is 1 byte) → there is a total of 300KB of allocated space for the entries
An evaluation will automatically convert to scientific notation if the result is either, between -9E-6 and 1E-6 → if the evaluation is greater than ~1.8E308 then a CalculationError will be thrown, or if it is between ±5E-324, then 0 will be returned
There may be cases of incorrect calculation and situations that were not tested → it is important to report any bugs or issues with the calculator so they can be fixed
The 'filter' section on the right of the answer entries page is used for finding and managing history entries→ the inputs are used to narrow down the filter (used for finding information) while the buttons are mostly for selection and deletion
The 'Select Mode' checkbox allows the entry to be selected on click while the 'regex' inputs are for regular expressions → regex is a mini-programming language used to match text based on a pattern, https://regexone.com/ is an example of a tutorial that may be useful
